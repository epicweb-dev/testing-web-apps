We want to test our onboarding process (it's one of the most important parts of
our app, so that's why we're giving it priority). Here are the current
instructions we have for the first part of our manual test:

1. Go to the homepage (<LinkToApp to="/" />)
2. Click the link that says <LinkToApp to="/login">Log in</LinkToApp>.
3. Verify the URL is <LinkToApp to="/login">`/login`</LinkToApp>.
4. Click the link that says <LinkToApp to="/signup">Create an account</LinkToApp>.
5. Verify the URL is <LinkToApp to="/signup">`/signup`</LinkToApp>.
6. Fill in the email field with a random email (ğŸ’° you can use [`faker`](https://fakerjs.dev/) if you like).

That's as far as we want you to go for this first test. Our tests go in the
`tests/e2e` directory. You'll start by creating a new file there called <InlineFile file="tests/e2e/onboarding.test.ts" />.

Once you've got that file created, go ahead and run the test script for E2E in UI Mode:

```sh
npm run test:e2e
```

If you'd rather simply run the test and get output in the terminal (without UI
mode), you can run:

```sh
npm run test:e2e:run
```
